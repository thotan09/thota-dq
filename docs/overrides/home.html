How it works

From business docs to diagnosed failures in minutes.

01
Point Aegis at your docs and warehouse
Give it your business docs — policies, schema definitions, SLAs, anything that describes your data. Aegis reads the context and generates data quality rules automatically. No hand-writing rules.
aegis generate orders --kb policy.md --kb schema.md
↓
02
Run checks across any warehouse
Aegis executes the rules against your warehouse. Every failure is classified by severity and diagnosed by an LLM — what broke, why it broke, and what regulation or SLA it violates.

Three ways to use Aegis

Generate
Rules from business docs
Point Aegis at your business docs and schema. It introspects the database, reads the context, and writes not_null, accepted_values, and complex custom_sql rules — including CTEs, window functions, and multi-table JOINs.
aegis generate transactions --kb policy.md --kb schema.md
Run
Pipeline manifests
A pipeline.yaml captures your database, rules, docs, LLM config, and goal in one file. Define it once. Run it from the CLI, Airflow, GitHub Actions, or any MCP client — same config everywhere.
aegis pipeline run my-project/pipeline.yaml
Talk
Conversational via Hermes
Connect Hermes via MCP. Ask it to load your pipeline, run validation, and diagnose failures in plain English. Hermes remembers past runs, can schedule recurring checks, and delivers results wherever you work.
Load the pipeline at demo/fraud/pipeline.yaml and run it.
